## Token Economics

### Minting Rewards (1:1 Ratio)
- 1 kWh solar generated = 1 ZSOLAR token
- 1 kWh battery discharged = 1 ZSOLAR token
- 1 kWh EV charging = 1 ZSOLAR token
- 1 mile EV driven = 1 ZSOLAR token

### Distribution on Mint
- 93% → User wallet
- 5% → Burned (deflationary)
- 1% → LP Rewards pool
- 1% → Treasury

### NFT Redemption
- Category NFTs (Solar, Battery, Charging, EV Miles) can be redeemed for tokens
- Redemption value = milestone threshold × 1 ZSOLAR
- 2% burn fee on redemption
- Welcome and Combo NFTs are NOT redeemable

### Transfer Tax (ERC-20)
- 3.5% burn on transfers
- 3.5% to treasury on transfers
- Exempt addresses: founder, owner, treasury, contract itself

## Metadata URI Structure

Base URI format: `ipfs://[CID]/`

Token URI: `ipfs://[CID]/[tokenId].json`

Example: Token ID 5 (Helios) → `ipfs://[CID]/5.json`

```json
{
  "name": "Helios",
  "description": "10,000 kWh generated",

  "attributes": [
    { "trait_type": "Category", "value": "Solar" },
```

## Key Notes

1. **Token IDs are 0-indexed** - Welcome = 0, first Solar = 1, etc.
2. **Fixed Token IDs** - Each milestone has a predetermined token ID that matches metadata
3. **One NFT per milestone per user** - Users cannot earn duplicate milestone NFTs
4. **Welcome NFT is unique** - Only one Welcome NFT exists (Token ID 0)
5. **Combo NFTs** are earned based on category NFTs only (combos don't count toward combos)
6. All thresholds are **greater than or equal to** (>=) checks
7. Each milestone mints exactly one NFT with a fixed token ID
